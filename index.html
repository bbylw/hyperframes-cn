核心优势

为什么选择 HyperFrames

HTML 原生

合成作品就是带有数据属性的 HTML 文件。无需学习 React，无需专有 DSL，无需复杂的构建流程。你已经会写 HTML，你已经准备好了。

data-composition-id · data-start · data-duration · data-track-index

AI 优先设计

智能体天生就懂 HTML。CLI 默认非交互式，专为 AI 驱动的工作流设计。用自然语言描述你要的视频，智能体帮你搞定一切。

确定性渲染

相同输入 = 相同输出。每一帧都精确可控，专为自动化流水线和 CI/CD 打造。告别随机性，拥抱确定性。

帧适配器模式

自带动画运行时集成——GSAP、Lottie、CSS 动画、Three.js，开箱即用。统一的帧适配器接口，让你在不同的动画引擎之间无缝切换。

三步创作视频

01
01

初始化项目

$ npx hyperframes init my-video

一行命令创建项目脚手架，自动安装依赖和 AI 技能。

02
02

实时预览

$ npx hyperframes preview

在浏览器中即时预览你的合成作品，支持实时重载。

03
03

渲染输出

$ npx hyperframes render

一键渲染为高质量 MP4，支持本地和 Docker 环境。

架构

模块化包架构

7 个独立包，各司其职，从核心类型到完整的浏览器编辑器。

CLI
hyperframes 主包

创建、预览、检查 lint 和渲染合成作品
核心
@hyperframes/core

类型、解析器、生成器、linter、运行时、帧适配器
引擎
@hyperframes/engine

可寻址页面到视频捕获引擎（Puppeteer + FFmpeg）
制作
@hyperframes/producer

完整渲染流水线（捕获 + 编码 + 音频混合）
工作室
@hyperframes/studio

基于浏览器的合成作品编辑器 UI
播放
@hyperframes/player

可嵌入的 <hyperframes-player> Web 组件
着色
@hyperframes/shader-transitions

用于合成作品的 WebGL 着色器过渡
